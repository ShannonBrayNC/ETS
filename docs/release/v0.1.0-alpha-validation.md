## Known release limitations

- ETS v0.1.0-alpha is for local validation, protocol review, and controlled demos.
- ETS stores metadata and hashes, not raw evidence bytes.
- Local unsigned tree heads are not production trust anchors.
- Local header and API-key auth modes are for development or demos only.
- Hosted deployments should use the production JWT/JWKS profile.
- SQLite is acceptable for local validation and demos, not final hosted production storage.
- RC consistency proof behavior is documented as alpha behavior and should not be treated as final production-grade consistency auditing.
- Production key discovery, rotation automation, multi-tenant operations, and hosted storage hardening remain future work.
