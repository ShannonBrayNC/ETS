# ETS v0.1.0-alpha Release Notes

Release status: draft until PR #15 is mergeable, validated, and merged to `main`.

## What ETS is

ETS, the Evidence Transparency System, is a reference transparency-log implementation for creating verifiable receipts over evidence metadata and artifact hashes.

ETS is designed to prove that metadata for a record, artifact, analysis, release, or workflow event was registered in a log and can later be verified against inclusion proofs and certificates.

## What this alpha includes

- Stable `EvidenceEvent` metadata contract
- Canonical JSON hashing
- In-memory append-only log
- SQLite persistence mode for local validation and demos
- Merkle inclusion proof generation and verification
- Proof bundle model and verifier helpers
- Certificate generation in JSON, Markdown, and HTML-oriented formats
- FastAPI API surface for event registration, proof generation, verification, certificates, metrics, and readiness checks
- Verifier CLI and SDK helpers
- Explorer UI reference workflow
- Local API request examples
- Fictional demo event and certificate samples
- Hosted JWT/JWKS auth profile documentation
- Tree-head signing guidance
- Scenario coverage for ten ETS product-fit workflows

## Trust boundaries

This alpha is not production trust infrastructure.

Important limitations:

- ETS stores metadata and hashes, not raw evidence bytes.
- Local unsigned tree heads are not production trust anchors.
- Local header and API-key auth modes are for development and demos only.
- Hosted deployments should use the production JWT/JWKS profile.
- SQLite is suitable for local validation and demos, not final hosted production storage.
- RC consistency proof behavior is documented as alpha behavior and should not be treated as final production-grade consistency auditing.
- Production key discovery and rotation automation are future work.
- Multi-tenant hosted operations require additional hardening.
